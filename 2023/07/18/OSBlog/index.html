哈工大 李治军网课学习笔记

02

应用通过操作系统的接口使用硬件

操作系统提供的接口:open,read,fork等

开机时发生了什么事情:

  1. 打开电源,计算机开始工作
  2. 计算机如何工作:取指执行

通用图灵机:

  1. 设置控制器动作(CPU，处理逻辑,应用程序)
  2. 控制器状态
  3. 数据

首先将程序加载进内存，使用PC(Program Counter)指针指向它，然后取指执行

X86PC:

(1)x86刚开机时CPU处于实模式

(2)开机时，CS = 0xFFFF，IP=0x0000CS：代码端寄存器，IP偏移

(3)寻址0xFFFF0(ROM BIOS映射区,BIOS:base input output system)，为什么是0xFFFF0，CS左移4位+IP。唯一有代码的地方

(4)检查RAM，键盘，显示器，软硬磁盘

(5)将磁盘0磁道0扇区(1个扇区512字节,00是操作系统的引导扇区)读入0x7c00处

(6)设置cs=0x07c0,IP=0x0000

引导扇区代码bootsect.s(汇编代码)

POSIX(Portable Operating System Interface of Unix)

系统调用的实现

用户为什么不能直接在内存中读取数据直接对数据进行操作，不能直接访问操作系统中的数据，尽管在同一内存中。因为root用户的密码。

硬件区分了内核态和用户态。内存被硬件划分位用户态和内核段。0：内核态，3是用户态
