STL

C++ STL

STL(Standard Template Library):标准模板库

标准库以header files形式呈现：

  • C++标准库的header files不带副档名(.h):#include < vector>
  • 新式C header files 不带副档名.h:#include
  • 旧式C header files 带有副档名.h,仍可用:#incldue < stdio.h >
  • 新式headers内的组件封装于namespace std
    • using namespace std::
    • using std::cout
  • 旧式headers内的组件不封装于namespace std;

STL六大组件:

  • 容器(Containers)
  • 算法(Algorithms)
  • 迭代器(Iterators)
  • 仿函数(Functors)
  • 适配器（配接器)(Adapters)
  • 空间配置器(Allocators)

STL六大组件的交互关系:

1.容器通过空间

2.算法通过迭代器存储容器中的内容

3.仿函数可以协助算法完成不同的策略的变化

4.适配器可以修饰仿函数

容器：各种数据结构，如vector、list、deque、set、map等，用来存放数据，从实现角度来看，STL容器是一种class template。(数据)

算法：各种常用的算法，如sort、find、copy、for_each。从实现的角度来看，STL算法是一种function tempalte。(操作)

迭代器：扮演了容器与算法之间的胶合剂，共有五种类型，从实现角度来看，迭代器是一种将operator*,operator->,operator++,operator–等指针相关操作予以重载的class template。(泛化的指针)

仿函数：行为类似函数，可作为算法的某种策略。从实现角度来看，仿函数是一种重载了operator()的class或者class template。(作用像个函数)

适配器：一种用来修饰容器或者仿函数或迭代器接口的东西。STL提供的queue和stack,虽然看似容器，但其实只能算是一种容器适配器，因为它们的底部完全借助deque,所有操作都由底层的deque供应。()

空间配置器：负责空间的配置与管理。从实现角度看，配置器是一个实现了动态空间配置、空间管理、空间释放的class template。

一般的分配器的std::alloctor都含有两个函数allocate与deallocte,这两个函数分别调用operator new()与delete(),这两个函数的底层又分别是malloc()和free()，但是每次malloc会带来格外开销(每次malloc一个元素都要带有附加信息)

STL优点：1.高可重用性 2.高性能 3.高移植性 4.跨平台 5.数据和操作分离

模板

因为STL使用了大量的模板，所以，在这里对模板进行一些列的讲解

定义

建立一个通用函数或类，其类内部的类型和函数的形式参数类型不具体指定，用一个虚拟的类型来表示。这种通用的方式称为模板。模板是泛型编程的基础，泛型编程即以一种独立于任何特定类型的方式编写代码。

一般使用

函数模板

建立一个通用函数，其函数类型和形参类型不具体指定，用一个虚拟的类型来表示。这个函数称为函数模板。

凡是函数体相同的函数都可以用这个模板来代替，不必定义多个函数，只需在模板中定义一次即可。在调用函数时系统会根据实参的类型来取代模板中的虚拟类型，从而实现了不同函数的功能

**函数模板定义形式:**模板说明+函数定义+函数模板调用

  • 类型形式参数表作为返回值时，需注意返回的类型

空间配置器(Allocator)

空间配置器总是隐藏在容器(conatiner)的背后。为什么是空间配置器而不是内存配置器，因为空间不一定是内存，空间也可以是磁盘或其他辅助存储介质。是的，可以写一个allocator直接向硬盘取空间。

主要作用:为容器分配内存空间，容器在实现中需要动态地分配和释放内存，分配器提供了一种统一的接口和策略来管理这些操作。负责跟踪可用内存并提供对分配和释放操作的支持

如何设计:

  • 向system heap要求空间
  • 考虑多线程(mutil-threads)状态
  • 考虑内存不足时的应变措施
  • 考虑过多”小性区块”可能造成的内存碎片(fragment)问题

考虑小型区块可能造成的内存破碎问题，SGI设计了双层级配置器，第一级直接使用malloc()和free()，第二级配置器则视情况采取不同的选择

SGI为alloc包装了一个接口:

第一级配置器以malloc(),free(),realloc()等C函数执行实际的内存配置、释放、重配置操作，并实现出类似C++ new-handler的机制(内存分配异常的处理机制。并非使用::operator new来配置内存。

C++ new-handler:要求系统再内存配置需求无法被满足时，调用一个你所指定的函数。**::operator new无法完成任务，再丢出std::bad_alloc 异常状态之前，会先调用由客端指定的处理例程。**C++并未提供相应于realloc()内存配置操作

第二级配置器 __default_alloc_template解析

第二级配置器多了一些机制，避免太多小额区块造成内存的碎片。小额区块带来的其实不仅是内存碎片，配置时的额外负担也是大问题。额外区块永远无法避免，毕竟系统要靠这多出来的空间管理内存。但是区块越小，额外负担所占的比例旧越大，越浪费。

第二级配置器的做法:如果区块够大，超过128bytes,就移交第一级配置器处理。当区块小于128bytes，则以内存池(memory pool)管理。

空间配置函数allocate()

身为一个配置器,__default_alloc_template拥有配置器的标准接口函数allocate()。此函数首先判断区块大小，大于128就调用第一级配置器，小于128bytes就检查对应的gree list。如果freelist之内有可用的区块，就直接用，没有就上调8倍数边界，调用refill()

vector

vector概述

vector的数据安排和操作方式与array相似。

array是静态空间，一旦配置的就不能改变，要用户手动扩展；vector是动态空间，会自行扩充空间。

vecotr定义摘要

vector的迭代器

vector维护的是一个连续线性空间，所有不论其元素型别如何，普通指针都可以作为vector的迭代器而满足所有必要条件

vector的插入可能导致迭代器失效,或者说有时部分失效

vector迭代器要执行的操作行为:

  • operator*,operator->,operator++,operator–,operator+,operator-,operator+=,operator-=

提供线性存储(Random Access Iterators)

vector的数据结构

vector采用的数据结构非常简单:线性连续空间。以三个迭代器来配置连续空间中的使用范围。vector的三个迭代器:start,finish,end_of_storage分别指向空间的头，已用空间的尾，空间的尾

finish是最后一个元素的结束位置，last+1的起始位置

operator[3]也是[3]号元素起始位置

vector的构造与内存管理:constructor,push_back

当我们以push_back()将新元素插入于vector尾端时，会先检查是否还有备用空间，如果有，直接在备用空间上构造元素，并调整迭代器finish,使vector变大。如果没有空间，就扩充空间。

动态增加大小，并不是在原空间后新增空间，而是以原大小的两倍另外配置一块空间，然后拷贝过来。

vector的元素操作:pop_back,erase,clear,insert

list

list概述

list的优点:每次插入或删除一个元素，旧配置或释放一个元素空间，对空间运用精准。

list的节点(node)

list本身和list的节点是不同的接口，需要分开设计

list的迭代器

list的迭代器相较于vector，更加地复杂。迭代器必须具备前移、后移的能力，所以list提供的是Bidirectional Iterators

list的插入(insert)和接合(splice)操作都不会造成原有的list迭代器失效。

list的数据结构

list不仅是一个双向链表，而且还是一个环状双向链表，所以它只需要一个指针，便可完整表现整个链表

如果指针node指向刻意置于尾端的一个空白节点，node便能符号STL的”前闭后开”的区间要求，成为list迭代器

list的构造与内存管理:constructor,push_back,insert

list缺省使用alloc作为空间配置器，并据此另外定义了一个list_node_allocator,为的是更方便地以节点大小为配置单位:

list的元素操作

list提供了很多元素操作:

  • push_front,push_back,erase,pop_front,pop_back,clear,remove,unique,splice,merge,reverse,sort

list内部提供了一个迁移操作(transfer):将某连续范围的元素迁移到某个特定位置之前。

deque

deque概述

deque是双向开口的连续线性空间。

deque允许在常数时间内对头端进行元素的插入或删除。deque没有capacity观念，因为它是动态地以分段连续空间组合而成，没有必要提供空间保留(reserve)功能。提供了Ramdom Access Iterator的接口。

deque的中控器

deque用复杂的迭代器架构，避免“重新配置、复制、释放”的操作

deque采用map(并不是STL的map容器)作为主控，map是一小块连续空间，其中每个元素都是指针，指向另一段连续线性空间。

deque的迭代器

deque是分段连续空间，维持其”整体连续的假象”，靠operator++,operator–

deque必须能够指出分段连续空间在哪，其次它必须能够判断自己是否已经处于所在缓冲区边缘，如果是，一旦前进或后退时就必须跳跃至下一个或上一个缓冲区。

假设一个deque< int >,令其缓冲区大小为32，于是每个缓冲区可容纳32/sizeof(int)= 4个元素。

deque的数据结构

deque除了维护一个map指针外，也维护start,finish两个迭代器，分别指向第一个缓冲区的第一个元素和最后缓冲区的最后一个元素。

deque的构造与内存管理

  • ctor,push_back,push_front

deque的元素操作

  • pop_back,pop_front,clear,erase,insert

pop_back,pop_front与之前的push_bakc类似，clear清除全部元素，但应留下一个默认缓冲区

stack

stack是一种先进后出(First In Last Out,FILO)的数据结构，它只有一个出口。

stack允许新增数据，移除元素，取得最顶端元素.

stack定义完整列表

以某种容器作为底部结构，将器接口改变，符合”先进后出”的特性,deque就可以实现这种结构

由于stack是以deque为底部容器完成其所有工作，而具有这种”修改某物接口，形成另一种风貌”，这种称为adapter(配置器)

stack没有迭代器

stack所有元素的进出都必须符合“先进后出”的条件，只有stack顶端的元素，才有机会被外界取用。stack不提供走访功能，也不提供迭代器。

以list作为stack的底层容器

list也是双向开口的数据接口,所有也可以作为底部结构。

queue

queue是一种先进先出(First In First Out,FIFO)的数据结构，有两个出口。queu允许新增元素，移除元素，从最底端加入元素，取得最顶端元素。

queue定义完整列表

以deque为底端实现queue。

queue没有迭代器

queue不提供遍历功能，也不提供迭代器

heap

heap并不是STL容器组件，是priority queue的助手。priority queue允许用户以任何次序将任何元素推入容器内，但取出数一定是从优先级最高的元素开始取。

complete binary tree整棵树内没有任何节点漏洞，可以用array来储存所有节点。将array的#0元素保留(可为无限大或无限小值)

**需要的工具:**一个vector和一组heap算法

heap可用max-heap或min-heap

priority_queue

priority_queue是一个拥有权值的queue，允许加入新元素、移除旧元素、审视元素值等功能，由于是queue，所以只允许在底端加入元素，从顶端取出元素。

关联式容器(associative containers)

标准的STL关联式容器分为set(集合)和map(映射)，以及这两大类的衍生体multiset和multimap。这些容器的底层机制均以RB-tree完成

**关联式容器(associative containers):**每笔数据都有一个键值(key)和一个实值(value)。当元素被插入到关联式容器中时，容器内部结构便依照其键值大小，以某种特定规则将这个元素放于适当位置。

关联式容器内部一般为balanced binary tree

树的导览

树(tree)是十分基础的数据结构。

二叉搜索树(binary search tree)

二叉搜索树(binary search tree)，可提供对数时间(log)的元素插入和访问。

**规则:**任何节点的键值一定大于其左子树中的每一个节点的键值，并小于其右子树中的每一个节点的键值。

平衡二叉搜索树(balanced binary search tree)

**平衡:**左右高度差为1

任何一个节点的左子树或者右子树都是「平衡二叉树」（左右高度差小于等于 1）

AVL tree(Adelson-Velskii-Landis tree)

AVL tree是一个“加上了额外平衡条件”的二叉搜索树。

平衡被破环的4种情况:

  • 插入点位于X的左子节点的左子树
  • 插入点位于X的左子节点的右子树
  • 插入点位于X的右子节点的左子树
  • 插入点位于X的右子节点的右子树

单旋转(Single Rotation)

A子树成长了一层，导致不平衡。B子树不可能和A子树位于同一层，否可k2再插入前就处于不平衡状态了。B子树也不可能和C子树位于同一层，否则第一个违反平衡条件的是k1而不是k2

**解决方法:**A提高一层，C子树下降一层。k1向上提，k2向下滑，并使B子树挂到k2的左侧。
