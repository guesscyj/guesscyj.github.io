c++相关学习笔记

为什么学习c++,c++难学易用，拥有广博的语法，能够实现各种各样的东西,巨大而变化多端的设计可以表现出来，同时也有可能导致你的代码难以理解、不易维护、不易扩充等。本文章基于《Effective c++》，《Effective Modern c++》等各类数据及网络文章学习总结而得。

2.explicit关键字和强制类型转换

explicit使隐式申明不能调用。在C++中，explicit 是一种关键字，用于修饰类的构造函数。如果一个构造函数被 explicit 修饰了，就表示该构造函数只能以显式的方式进行调用，不能隐式地自动转换为其它类型，从而避免了隐式类型转换可能带来的问题。

使用 explicit 关键字修饰构造函数，可以确保该函数只能以显式的方式调用。这样可以避免隐式类型转换可能带来的歧义和错误，提高程序的可读性和安全性。

3.将c++视为语言联邦，而非单一语言

c++一开始的名称为c with classes，随着后续的更新，越来越成熟，引入了各种各样的特性，c++有4个主要部分

  • C
  • Object-Oriented C++
  • Template C++
  • STL

c++以c为基础，拓展了许多的功能。Object-Oriented c++为这门语言带来了class、encapsulation（封装）、inheritance(继承)、polymorphism(多态)、virtual(动态绑定)。Template C++为这门语言带来了generic programming(范型编程)，为代码的重用提供了基础。STL是个template程序库，对contaimers(容器)、iterators(迭代器)、algorithms(算法)、function objects(函数对象)的规约有极佳的紧密配合与协调。

5.尽量使用const

use const whenever possible

当一个对象不该被改动时，就该对其进行const声明，通过编译器来进行保证。

一些能使用const修饰的内容

const关键字出现在星号左边，表示被指物(内存地址)是常量；出现在星号右边，表示指针为常量。

const最具威力的用法是面对函数声明时的应用。在一个函数声明式内，const可以和函数返回值、各参数、函数本身产生关联。

令函数返回一个常量值，往往可以降低因客户错误而造成的意外，而又不至于放弃安全性和高效性

可以防止出现这种代码编写错误，防止出现无意义的赋值，使class接口比较容易理解，哪个函数可以改动对象内容，使“操作const”对象成为可能。

成员函数后面的 const 修饰符可以用于指定该成员函数是不会修改对象的状态的（成员变量），也就是说该函数不会对对象进行写操作。使用 const 修饰符可以使得编译器对这个成员函数做出一些特殊的优化，同时也可以保证程序的安全性。对成员变量添加mutable进行修饰，之后便可以在const成员函数中进行修改

non-local static对象的初始化次序:

static对象：其寿命从构造到程序结束，因此stack和heap-based对象都将被排除

包括golbal对象、定义于namespace作用域内的对象、在classes内、在函数内、以及在file作用域内被声明为static的对象。函数内的static对象称为local static对象，其他static对象称为non-local static对象。程序结束时static对象将会自动销毁。

**编译单元(translation unit)**是指产出单一目标文件(single object file)的源码。它基本上是单一源码文件加上所含入的头文件(#include files)。

如果某一个non-local static对象使用了另一个non-local static对象,但该对象为先初始化，则会造成无法建立。为什么不规定non-local static对象的次序？绝对初始化顺序非常困难。

将每个non-local static对象放置在自己专属的函数内(在函数内声明为static)，将non-local static对象使用local static替换。单例模式。

10.别让异常逃离析构函数

Prevent exceptions from leavind destructors

在析构v时，假如调用Widget的析构函数有一个异常抛出，然后又一个异常抛出，此时程序将会又不明确的行为。

但在析构函数中进行操作不好，应该避免在析构函数中抛出异常。

  • 析构函数绝对不要吐出异常。如果一个被析构函数调用的函数可能抛出异常，析构函数应该捕捉任何异常，然后吞下异常或结束程序
  • 如果客户需要对某个操作函数运行期间抛出的异常做出反应，那么class应该提供一个普通函数来代替执行

Never call virtual functions during construction or destruction

BuyTransaction的构造函数被调用，但由于其继承了Transaction，故Transaction的构造函数先调用。此时，base class的构造函数将调用logTransaction（Transaction的），对，没调用derived class 的对应版本，virtual函数没有下降到derived class层，在base class构造期间，virtual函数不是virtual函数。程序可能无法连接。

可能这种操作很反直觉，但是好的行为，因为derived class在次期间中，成员变量几乎都没初始化。当然，运行期类型信息(runtime type information,如dynamic_cast和typeid)也是同样的效果。

同理，析构函数开始执行，derived class变量也会呈现为未定义值。

logTransaction如果是纯虚函数，会终止调用，结束程序（毕竟pure virtual),但如果是impure virtual,会执行一个错误版本的logTransaction,还不会报错。

由于无法使用virtual函数从base classes向下调用，在构造期间，令derived classes将必要的构造信息传递至base class构造函数来替代。

  • 在构造和析构期间不要调用virtual函数，因为这类调用从不下降至derived class(当然，也是为了保证程序的正确性)。

13.多态和继承

多态性指的是在面向对象程序设计中，一个对象可以表现出多种类型和行为的能力。具体而言，它包括两个方面的含义：静态多态和动态多态。

静态多态通常通过函数重载和运算符重载来实现，同一函数或运算符可以接受不同的参数类型，从而实现不同的行为。

动态多态则是基于继承和虚函数的概念实现的。在这种情况下，派生类可以重写基类的虚函数，从而在调用时产生不同的行为。通过基类指针或引用来调用虚函数时，会根据指向的对象类型动态地选择要调用的函数版本，这样可以实现面向对象编程中的“接口规范化”和“多态替换”。

多态性的使用可以使程序更加灵活，代码的可重用性和可扩展性也得到了提升。

面向对象编程中的继承是指一个类（子类）可以从另一个类（父类或基类）继承其属性和方法。继承允许您创建一个新类，它可以从现有类中继承属性和行为，并且还可以添加自己的属性和行为。继承关系通常表示“is-a”关系，即子类是父类的一种类型。

在 C++ 中，使用关键字 class 或 struct 来定义类，派生类可以通过 class B : public A 的形式来继承基类 A。继承后，子类可以使用父类的公共成员函数和变量，也可以重写（override）父类中的成员函数，实现多态性。

有以下几种继承方式：

  1. 公有继承：派生类可以继承父类的公有成员，但不能继承私有成员。
  2. 私有继承：派生类可以继承父类的公有和保护成员，但都会变成私有成员。
  3. 保护继承：派生类可以继承父类的公有和保护成员，但都会变成保护成员。

继承是面向对象编程中重要的概念之一。它能够使代码结构更加清晰、易于理解和维护，同时也提高了代码的可重用性和可扩展性。

14. struct atomic，线程安全为什么这么重要
