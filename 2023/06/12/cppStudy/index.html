Effective C++

为什么学习c++,c++难学易用，拥有广博的语法，能够实现各种各样的东西,巨大而变化多端的设计可以表现出来，同时也有可能导致你的代码难以理解、不易维护、不易扩充等。本文章基于《Effective c++》，《Effective Modern c++》等各类数据及网络文章学习总结而得。

1.将c++视为语言联邦，而非单一语言

c++一开始的名称为c with classes，随着后续的更新，越来越成熟，引入了各种各样的特性，c++有4个主要部分

  • C
  • Object-Oriented C++
  • Template C++
  • STL

c++以c为基础，拓展了许多的功能。Object-Oriented c++为这门语言带来了class、encapsulation（封装）、inheritance(继承)、polymorphism(多态)、virtual(动态绑定)。Template C++为这门语言带来了generic programming(范型编程)，为代码的重用提供了基础。STL是个template程序库，对contaimers(容器)、iterators(迭代器)、algorithms(算法)、function objects(函数对象)的规约有极佳的紧密配合与协调。

Use objects to manage resources.

15.在资源管理类中提供原始资源的访问

  • 对原始资源的访问可能经由显式或隐式转换。一般而言显式更安全，隐式更方便。

  2. 由于TMP执行于C++编译期，因此可将工作从运行期转移到编译期。

  2. TMP的C++程序更高效:较小的可执行文件，较短的运行期，较少的内存需求。但编译时间更长。

  • 优化矩阵运算
  • 可以生成客户定制之设计模式实现品

总结:

  • 可将工作由运行期移到编译期，因而得以实现早期错误侦测和更高的执行效率
  • TMP可被用来生成基于政策选择组合的客户定制代码，也可用来避免生成对某些特殊类型并不适合的代码

49.了解new-handler的行为

Understand the behavioe of the new-handler.

当operator new抛出异常以反映一个未获满足的内存需求之前，它会先调用一个客户指定的错误处理函数，一个所谓的new-handler。

当operator new无法满足内存申请时，它会不断调用new-handler函数，直到找到足够内存。

  • 设置new-handler:可以使用std::set_new_handler函数来设置new_handler。这个函数接受一个指向函数的指针。
  • 触发new-handler:当默认的operator new无法满足内存分配请求时，会调用与new-handler关联的函数。
  • 默认的new-handler行为:如果未设置自定义的new-handler，而又无法满足内存分配请求，抛出bad_alloc异常

良好的new-handler函数设计:

  • 让更多内存可被使用：预分配，先分配一块内存，当new-handler第一次被调用，将内存释放给程序使用。
  • 安装另一个new-handler：new-handler修改“会影响new-handler行为”的static数据、namespace数据或global数据。
  • 卸除new-handler：将null指针传给set_new_handler。一旦没有安装任何new-handler，operator new会在内存分配不成功时抛出异常。
  • 抛出bad_alloc(或派生自bad_alloc)的异常：这样的异常不会被operator new捕捉，因此会被传播到内存索求处。
  • 不返回：通常调用abort或exit

有时候，希望能以不同的方式处理内存分配失败情况

C++并不支持class专属之new-handlers，也不需要。可以自己实现，只需令每一个class提供自己的set_new_handler和operator new即可。

  • 当你声明placement new和placement delete，请不要无意识(非故意)地遮掩它们的正常版本。

53.不要轻忽编译器的警告

Pay attention to compiler warnings.

不要忽视编译器的警告

  • 严肃对待编译器发出的警告信息。努力在你的编译器的最高警告级别下争取”无任何警告”的荣誉
  • 不要过度依赖编译器的报警能力，因为不同的编译器对待事情的态度并不相同。一旦移植到另一个编译器上，你原本依赖的警告信息有可能消失

54.让自己熟悉包括TR1在内的标准程序库

Familiarize yourself with the standard library,includeing TR1.

TR1(Technical Report 1)，是C++程序库工作小组对该份文档的称呼。(std::tr1::shared_ptr)

C++98列入C++标准程序库的主要成分:

  • STL(Stndard Template Library,标准模板库)，覆盖容器(containers)，迭代器(iterators)，算法(algorithms)，函数对象(function objects)，适配器(container adapters)和函数对象适配器(function object adapters)
  • Iostream，覆盖用户自定缓冲功能，国际化I/O，以及预先定义好的对象cin,cout,cerr,clog
  • 国际化支持
  • 数值处理，包括复数模板(complex)和纯数值数组(valarray)
  • 异常阶层体系(exception hierarchy)，包括base class exception及其derived classes logic_class和runtime_error，以及更深继承的各个classes
  • C89标准程序库

tr1组件:

  • 智能指针(smart pointers)：shared_ptr和weak_ptr
  • tr1::function,此物得以表示任何callable entity(可调用物，也就是任何函数或函数对象)，只要其签名符合目标。void registerCallback(std::string func(int));
  • tr1::bind:STL绑定器(binders)
  • Hash tables
  • 正则表达式(Regular expressions)
  • Tuples(变量组)
  • tr1::array
  • tr1::mem_fn
  • tr1::reference_wrapper
  • 随机数(random number)生成工具
  • 数学特殊函数
  • C99兼容扩充
  • Type traits
  • tr1::result_of

总结：

  • C++标准程序库的主要机能由STL、iostreams、locales组成。并包含C99标准程序库
  • TR1添加了智能指针、一般化函数指针、hash-based容器、正则表达式以及其他组件的支持
  • TR1自身只是一份规范。
