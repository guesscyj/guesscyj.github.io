SQL

SQL简介

SQL (Structured Query Language:结构化查询语言) 是用于管理关系数据库管理系统（RDBMS）

  • SQL 对大小写不敏感：SELECT 与 select 是相同的。

三大范式

1.第一范式(1NF):确保每列具有原子性，即每个单元格中的数据都是不可再分的。表中的每个字段都应该是原子的，不应该包含重复的数据

2.第二范式(2NF):要求满足第一范式，并且消除非主键列对于主键的部分依赖。每个非主键列必须完全依赖于主键，而不是依赖于主键的部分属性。

3.第三范式(3NF):要求满足第三范式，并且消除主键列之间的传递依赖。任何非主键列不能依赖其他非主键列。

4.巴斯-科德范式(BCNF):要求满足第三范式，并且消除主键依赖非主键的情况。任何主键都不能依赖非主键。

范式的级别越高，数据重复性越低，更新操作时的数据一致性和完整性也更容易维护。但过度范式化可能导致查询复杂度增加，因此在实际设计数据库时需要权衡范式和实际需求之间的关系。

需要注意的是，范式化只是数据库设计的一部分，具体的设计还需要考虑性能优化、数据访问模式等因素，以及根据具体业务需求进行适当的反范式化。

数据类型

  1. 整数类型(Integer Types):TINYINT、SMALLINT、MEDIUMINT、INT和BIGINT
  2. 浮点数类型(Floating-Point Types):FLOAT和DOUBLE
  3. 字符串类似(String Types):CHAR、VARCHAR、TEXT和BLOB
  4. 日期和时间类型(Date and Time Types): DATE、TIME、DATETIME和TIMESTAMP
  5. 布尔类型(Boolean Types):BOOL、BOOLEAN
  6. 枚举类型（Enumeration Types）：用于在几个预定义值中选择一个值进行存储。
  7. 集合类型（Set Types）：类似于枚举类型，但可以选择多个值进行存储。

SQL约束(CONSTRAINT)

SQL约束用于规定表中的数据规则。可在create或alter时规定

sql有以下约束:

  • NOT NULL:某列不能存储NULL值
  • UNIQUE:某列的每行必须有唯一的值
  • PRIMARY KEY:NOT NULL & UNIQUE
  • FOREIGN KEY:保证一个表中的数据匹配另一个表中的值的参照完整性
  • CHECK:保证列中的值符合指定的条件
  • DEFAULT:给列赋值时的默认值

SQL JOIN

用于把来自两个或多个表的行结合起来,基于这些表之间的共同字段。

最常见的JOIN类型：SQL INNER JOIN(简单的jpin)

JOIN是将两个或多个表中满足连接条件的行合并起来，如果某个表中没有匹配的行，则不会包含在结构集中，不会填写NULL

例子:

INNER JOIN:如果表中有至少一个匹配，则返回行

LEFT JOIN:即使右表中没有匹配，也从左表返回所有的行

RIGHT JOIN:即使左表中没有匹配，也从右表中返回所有的行

FULL JOIN:只要其中一个表中存在匹配，则返回行

LEFT JOIN

SQL Aggregate 函数计算从列中取得的值，返回一个单一的值。

有用的 Aggregate 函数：

  • AVG() - 返回平均值
  • COUNT() - 返回行数
  • FIRST() - 返回第一个记录的值
  • LAST() - 返回最后一个记录的值
  • MAX() - 返回最大值

SQL Scalar 函数

SQL Scalar 函数基于输入值，返回一个单一的值。

有用的 Scalar 函数：

  • UCASE() - 将某个字段转换为大写
  • LCASE() - 将某个字段转换为小写
  • MID() - 从某个文本字段提取字符，MySql 中使用
  • SubString(字段，1，end) - 从某个文本字段提取字符
  • LEN() - 返回某个文本字段的长度
  • ROUND() - 对某个数值字段进行指定小数位数的四舍五入
  • NOW() - 返回当前的系统日期和时间
  • FORMAT() - 格式化某个字段的显示方式

关系型和非关系型数据库的区别

关系型数据库(Relational Database)和非关系型数据库(Non-Relational Database)

关系型数据库:

  • 使用表格(即关系)来组织和存储数据，数据以行和列的形式进行表示
  • 每个表格由多个列构成，每个列代表一个数据字段
  • 表格之间可以建立关联关系，通过主键和外键来定义
  • 使用结构化查询语言进行数据操作和查询
  • 具备ACID(原子性、一致性、隔离性和持久性)
